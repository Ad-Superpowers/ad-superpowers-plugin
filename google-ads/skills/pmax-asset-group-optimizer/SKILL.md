## Asset Performance Ratings

### Understanding Asset Ratings

```
ASSET PERFORMANCE RATINGS EXPLAINED
═══════════════════════════════════

RATING SYSTEEM:
───────────────
┌─────────────┬─────────────────────────────────────────────────┐
│ Rating      │ Betekenis                                       │
├─────────────┼─────────────────────────────────────────────────┤
│ Best        │ Top performer, outperforms other assets         │
│             │ → Keep running, maak vergelijkbare variaties    │
├─────────────┼─────────────────────────────────────────────────┤
│ Good        │ Performs well, average of better                │
│             │ → Keep running, monitor voor changes            │
├─────────────┼─────────────────────────────────────────────────┤
│ Low         │ Underperforms vs andere assets                  │
│             │ → Vervang na 30 dagen consistent Low            │
├─────────────┼─────────────────────────────────────────────────┤
│ Pending     │ Nog onvoldoende data voor rating                │
│             │ → Wacht, geen actie nodig                       │
└─────────────┴─────────────────────────────────────────────────┘

WAAR TE VINDEN:
───────────────
Campaign → Asset Groups → [Asset Group] → Asset Performance

IMPORTANT NOTES:
────────────────
⚠️ Ratings zijn RELATIEF binnen jouw asset group
   "Low" betekent niet slecht, maar minder goed dan anderen

⚠️ Houd minimaal 3-5 assets van elk type
   Anders geen vergelijking mogelijk

⚠️ Wacht minimaal 2-3 weken voor betrouwbare ratings
   Te snel vervangen verstoort learning
```

### Asset Optimization Workflow

```
ASSET OPTIMIZATION CYCLE
════════════════════════

WEEK 1-2: LEARNING PHASE
────────────────────────
□ Upload alle assets (max aantallen)
□ Geen wijzigingen maken
□ Ratings nog "Pending" of instabiel
□ Focus op delivery en basics

WEEK 3-4: EERSTE ANALYSE
────────────────────────
□ Check asset ratings
□ Identificeer eerste "Low" performers
□ Nog geen verwijderingen
□ Noteer trends

WEEK 5-6: EERSTE OPTIMALISATIE
──────────────────────────────
□ Vervang assets die consistent "Low" zijn
□ Voeg variaties toe van "Best" assets
□ Test nieuwe concepten
□ Monitor impact

MAANDELIJKS: ONGOING OPTIMIZATION
─────────────────────────────────
□ Full asset review
□ Vervang 20-30% van assets (refresh)
□ Seasonal updates
□ A/B test nieuwe concepten

VERVANGINGSREGELS:
──────────────────
┌────────────────────────────────────────────────────────────────┐
│ Asset Rating │ Duration │ Action                              │
├──────────────┼──────────┼─────────────────────────────────────┤
│ Low          │ >30 days │ Vervang                             │
│ Low          │ <30 days │ Monitor, nog niet vervangen         │
│ Good         │ Any      │ Keep, optioneel test variaties      │
│ Best         │ Any      │ Keep, maak vergelijkbare variaties  │
│ Pending      │ Any      │ Wacht op meer data                  │
└──────────────┴──────────┴─────────────────────────────────────┘
```

## Asset Requirements & Best Practices

### Complete Asset Checklist

```
ASSET GROUP ASSET REQUIREMENTS
══════════════════════════════

HEADLINES (Verplicht)
─────────────────────
├── Minimum: 3 headlines
├── Aanbevolen: 5-15 headlines
├── Maximum: 15 headlines
├── Lengte: Max 30 karakters
└── Best practices:
    ├── Mix van USPs, CTAs, features
    ├── Include brand name in 1-2
    ├── Varieer in lengte (kort en lang)
    ├── Test cijfers/percentages
    └── Geen herhaling

LONG HEADLINES (Verplicht)
──────────────────────────
├── Minimum: 1 long headline
├── Aanbevolen: 3-5 long headlines
├── Maximum: 5 long headlines
├── Lengte: Max 90 karakters
└── Best practices:
    ├── Storytelling mogelijk
    ├── Meer context dan headlines
    ├── Include CTA
    └── Wordt gebruikt in grotere placements

DESCRIPTIONS (Verplicht)
────────────────────────
├── Minimum: 2 descriptions
├── Aanbevolen: 4-5 descriptions
├── Maximum: 5 descriptions
├── Lengte: Max 90 karakters
└── Best practices:
    ├── Eerste description = beste hook
    ├── Varieer: benefits, features, social proof
    ├── Geen herhaling van headlines
    └── Include urgency waar relevant

IMAGES (Verplicht)
──────────────────
├── Minimum: 1 landscape + 1 square
├── Aanbevolen: 3-5 van elk type
├── Maximum: 20 images totaal
├── Formaten:
│   ├── Landscape (1.91:1): 1200x628 aanbevolen
│   ├── Square (1:1): 1200x1200 aanbevolen
│   └── Portrait (4:5): 960x1200 aanbevolen
└── Best practices:
    ├── Mix product shots en lifestyle
    ├── Geen tekst op >20% van beeld
    ├── Hoge kwaliteit (geen blur)
    ├── Gezichten presteren goed
    └── Mobile-first design

LOGOS (Verplicht)
─────────────────
├── Square logo (1:1): Min 128x128, aanbevolen 1200x1200
├── Landscape logo (4:1): Min 512x128, aanbevolen 1200x300
└── Transparante achtergrond aanbevolen

VIDEOS (Optioneel maar aanbevolen)
──────────────────────────────────
├── Minimum: 0 (niet verplicht)
├── Aanbevolen: 1-3 videos
├── Maximum: 5 videos
├── Formaten:
│   ├── Landscape (16:9)
│   ├── Square (1:1)
│   └── Vertical (9:16) voor Shorts
├── Lengte: 10 sec - 3 min (15-30 sec aanbevolen)
└── Best practices:
    ├── Hook in eerste 3 seconden
    ├── Brand in eerste 5 seconden
    ├── Werkt zonder geluid (ondertiteling!)
    └── CTA visueel en verbaal

⚠️ GEEN VIDEO UPLOAD = GOOGLE MAAKT AUTO-VIDEO
   Google kan automatisch videos maken van je images.
   Dit is vaak minder effectief dan eigen videos.
```

### Audience Signal Troubleshooting

```
AUDIENCE SIGNALS ISSUES & FIXES
═══════════════════════════════

PROBLEEM: Signals worden genegeerd
──────────────────────────────────
Symptomen:
├── Insights tonen andere audiences
├── Geen delivery naar je signals
└── AI vindt andere audiences

Oorzaken & Oplossingen:
├── Signal lists te klein (<1000)
│   └─► Vergroot lists, combineer segments
├── Signals niet relevant voor product
│   └─► Review en update signals
├── AI vindt betere audiences
│   └─► Laat AI werken, monitor results
└── Optimized targeting te agressief
    └─► Overweeg audience-only targeting test

PROBLEEM: Te brede targeting
────────────────────────────
Symptomen:
├── Hoge CPM maar lage relevantie
├── Veel impressies, weinig conversies
└── Search Terms irrelevant

Oplossingen:
├── Meer first-party data toevoegen
├── Specifiekere custom segments
├── Exclude irrelevante audiences
└── Meer specific search themes

PROBLEEM: Te nauwe targeting
────────────────────────────
Symptomen:
├── Limited delivery
├── High CPMs door beperkte competitie
└── Learning phase duurt lang

Oplossingen:
├── Voeg bredere audiences toe
├── Verwijder overly specific signals
├── Verhoog budget voor learning
└── Enable optimized targeting
```

## Google Ads Script: Asset Group Monitor

```javascript
/**
 * Performance Max Asset Group Monitor
 *
 * Monitort PMax asset groups en stuurt
 * wekelijkse performance reports.
 *
 * Features:
 * - Asset group performance comparison
 * - Conversion/ROAS tracking per AG
 * - Trend analysis
 * - Recommendations
 *
 * Schedule: Wekelijks op maandag
 */

var CONFIG = {
  EMAIL: 'jouw@email.com',

  // Thresholds
  MIN_SPEND: 50,                 // Minimum spend voor analyse
  LOW_ROAS_THRESHOLD: 1.5,       // ROAS onder 1.5x = alert
  HIGH_CPA_MULTIPLIER: 1.5,      // CPA 1.5x boven gemiddelde

  // Date range
  DATE_RANGE: 'LAST_7_DAYS',
  COMPARE_RANGE: 'LAST_14_DAYS'
};

function main() {
  var assetGroupData = [];

  // Query asset group performance
  var query =
    'SELECT campaign.name, asset_group.name, asset_group.id, ' +
    'metrics.impressions, metrics.clicks, metrics.cost_micros, ' +
    'metrics.conversions, metrics.conversions_value ' +
    'FROM asset_group ' +
    'WHERE campaign.advertising_channel_type = "PERFORMANCE_MAX" ' +
    'AND segments.date DURING ' + CONFIG.DATE_RANGE;

  try {
    var report = AdsApp.report(query);
    var rows = report.rows();

    while (rows.hasNext()) {
      var row = rows.next();

      var costMicros = parseInt(row['metrics.cost_micros']) || 0;
      var cost = costMicros / 1000000;

      if (cost < CONFIG.MIN_SPEND) continue;

      var impressions = parseInt(row['metrics.impressions']) || 0;
      var clicks = parseInt(row['metrics.clicks']) || 0;
      var conversions = parseFloat(row['metrics.conversions']) || 0;
      var value = parseFloat(row['metrics.conversions_value']) || 0;

      assetGroupData.push({
        campaign: row['campaign.name'],
        assetGroup: row['asset_group.name'],
        impressions: impressions,
        clicks: clicks,
        cost: cost,
        conversions: conversions,
        value: value,
        ctr: impressions > 0 ? clicks / impressions : 0,
        cpc: clicks > 0 ? cost / clicks : 0,
        cpa: conversions > 0 ? cost / conversions : 0,
        roas: cost > 0 ? value / cost : 0,
        convRate: clicks > 0 ? conversions / clicks : 0
      });
    }
  } catch (e) {
    Logger.log('Query error: ' + e.message);
  }

  // Calculate averages for comparison
  var avgCPA = 0;
  var avgROAS = 0;
  var totalConv = 0;
  var totalCost = 0;
  var totalValue = 0;

  for (var i = 0; i < assetGroupData.length; i++) {
    totalConv += assetGroupData[i].conversions;
    totalCost += assetGroupData[i].cost;
    totalValue += assetGroupData[i].value;
  }

  avgCPA = totalConv > 0 ? totalCost / totalConv : 0;
  avgROAS = totalCost > 0 ? totalValue / totalCost : 0;

  // Identify issues
  var issues = [];
  var topPerformers = [];
  var lowPerformers = [];

  for (var j = 0; j < assetGroupData.length; j++) {
    var ag = assetGroupData[j];

    // Low ROAS
    if (ag.roas < CONFIG.LOW_ROAS_THRESHOLD && ag.conversions > 0) {
      lowPerformers.push({
        name: ag.assetGroup,
        campaign: ag.campaign,
        issue: 'Low ROAS: ' + ag.roas.toFixed(2) + 'x',
        cost: ag.cost,
        roas: ag.roas
      });
    }

    // High CPA
    if (ag.cpa > avgCPA * CONFIG.HIGH_CPA_MULTIPLIER && ag.conversions > 0) {
      issues.push({
        name: ag.assetGroup,
        campaign: ag.campaign,
        issue: 'High CPA: €' + ag.cpa.toFixed(2) + ' (avg: €' + avgCPA.toFixed(2) + ')'
      });
    }

    // Top performers
    if (ag.roas > avgROAS * 1.2 && ag.conversions >= 3) {
      topPerformers.push({
        name: ag.assetGroup,
        campaign: ag.campaign,
        roas: ag.roas,
        conversions: ag.conversions
      });
    }
  }

  // Sort
  assetGroupData.sort(function(a, b) { return b.value - a.value; });
  topPerformers.sort(function(a, b) { return b.roas - a.roas; });
  lowPerformers.sort(function(a, b) { return a.roas - b.roas; });

  // Send report
  sendReport(assetGroupData, topPerformers, lowPerformers, issues, avgCPA, avgROAS);
  Logger.log('Asset Group Monitor complete. Groups analyzed: ' + assetGroupData.length);
}

function sendReport(data, top, low, issues, avgCPA, avgROAS) {
  if (data.length === 0) {
    Logger.log('No asset groups with sufficient data');
    return;
  }

  var subject = '📊 PMax Asset Group Report - ' + AdsApp.currentAccount().getName();

  var body = 'Performance Max Asset Group Report\n';
  body += '===================================\n\n';
  body += 'Account: ' + AdsApp.currentAccount().getName() + '\n';
  body += 'Period: ' + CONFIG.DATE_RANGE + '\n';
  body += 'Asset Groups Analyzed: ' + data.length + '\n\n';

  // Summary
  var totalSpend = 0;
  var totalConv = 0;
  var totalVal = 0;
  for (var i = 0; i < data.length; i++) {
    totalSpend += data[i].cost;
    totalConv += data[i].conversions;
    totalVal += data[i].value;
  }

  body += 'SUMMARY\n';
  body += '───────\n';
  body += 'Total Spend: €' + totalSpend.toFixed(2) + '\n';
  body += 'Total Conversions: ' + totalConv.toFixed(1) + '\n';
  body += 'Average CPA: €' + avgCPA.toFixed(2) + '\n';
  body += 'Average ROAS: ' + avgROAS.toFixed(2) + 'x\n\n';

  // Top Performers
  if (top.length > 0) {
    body += '⭐ TOP PERFORMERS (' + top.length + '):\n';
    body += '──────────────────────────\n';
    for (var t = 0; t < Math.min(5, top.length); t++) {
      body += '• ' + top[t].name + '\n';
      body += '  ROAS: ' + top[t].roas.toFixed(2) + 'x | Conv: ' + top[t].conversions.toFixed(1) + '\n';
    }
    body += '\n';
  }

  // Low Performers
  if (low.length > 0) {
    body += '⚠️ NEEDS ATTENTION (' + low.length + '):\n';
    body += '──────────────────────────────\n';
    for (var l = 0; l < Math.min(5, low.length); l++) {
      body += '• ' + low[l].name + '\n';
      body += '  ' + low[l].issue + '\n';
    }
    body += '\n';
  }

  // All Asset Groups
  body += 'ALL ASSET GROUPS (by value):\n';
  body += '────────────────────────────\n';
  for (var d = 0; d < data.length; d++) {
    var ag = data[d];
    body += '• ' + ag.assetGroup + ' (' + ag.campaign + ')\n';

    body += ' | Conv: ' + ag.conversions.toFixed(1);
    body += ' | ROAS: ' + ag.roas.toFixed(2) + 'x\n';
  }

  body += '\n---\nGenerated by Asset Group Monitor';

  MailApp.sendEmail(CONFIG.EMAIL, subject, body);
}
```
