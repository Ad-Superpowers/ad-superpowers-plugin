## Quick Decision Guide

```
LEARNING PHASE STATUS - WHAT TO DO NOW?
│
├─► STATUS: "Learning"
│   └─► WAIT AND OBSERVE
│       ├── Normal duration: 7-14 days
│       ├── DO NOT make changes
│       ├── Monitor daily, do not intervene
│       └── Expect fluctuations in CPA/ROAS
│
├─► STATUS: "Learning (limited)"
│   └─► ACTION REQUIRED
│       ├── Check: Sufficient budget?
│       ├── Check: Sufficient conversions?
│       ├── Check: Tracking issues?
│       └── See: "Resolving Learning Limited"
│
├─► STATUS: "Eligible"
│   └─► LEARNING COMPLETE
│       ├── You can now adjust targets
│       ├── Start with small changes (10%)
│       ├── Monitor performance
│       └── Optimize if stable
│
└─► STATUS: "Limited" (not learning-related)
    └─► OTHER LIMITATION
        ├── Budget limited?
        ├── Bid limited?
        ├── Target too restrictive?
        └── See: "Bid Status Issues"
```

## Resolving Learning Limited

### Diagnosis

```
"LEARNING (LIMITED)" DIAGNOSIS
═════════════════════════════

STEP 1: IDENTIFY THE CAUSE
────────────────────────────
Hover over the status for details, or check:

MOST COMMON CAUSES:

1. INSUFFICIENT CONVERSIONS
   └── Symptoms: <50 conversions in 30 days
   └── Impact: AI cannot find a pattern
   └── Solution: Increase volume or move up-funnel

2. BUDGET TOO LOW
   └── Symptoms: Budget limited at the same time
   └── Impact: Not enough auctions to learn from
   └── Solution: Increase budget or consolidate

3. TARGET TOO RESTRICTIVE
   └── Symptoms: Very low impressions
   └── Impact: Too few eligible auctions
   └── Solution: Relax the target

4. TRACKING ISSUES
   └── Symptoms: Conversions fluctuate wildly
   └── Impact: Unreliable data
   └── Solution: Fix tracking first

5. SEASONALITY/VOLATILITY
   └── Symptoms: Large day-to-day variation
   └── Impact: No consistent pattern
   └── Solution: Wait or use seasonality rules
```

### Solutions

```
LEARNING LIMITED SOLUTIONS
════════════════════════════

SOLUTION 1: INCREASE CONVERSION VOLUME
──────────────────────────────────────
Options:
├── A) Increase budget → more impressions → more conversions
├── B) Broaden targeting → more reach
├── C) Use "lighter" conversion (temporarily)
│   └── Form view instead of submit
│   └── Add-to-cart instead of purchase
│   └── Page engagement instead of lead
└── D) Combine campaigns in a portfolio

When using option C: Ensure the "real" conversion is still tracked
   for later use as primary action.

SOLUTION 2: INCREASE BUDGET
───────────────────────────
Calculation:
├── Target CPA x 10 = minimum daily budget
├── Example: $50 CPA → $500/day minimum

Or:
├── Combine budgets from multiple campaigns
├── Pause underperforming campaigns
├── Reallocate to learning campaigns

SOLUTION 3: RELAX THE TARGET
───────────────────────────
If tCPA is too aggressive:
├── Increase tCPA by 20-30%
├── Or switch to Max Conversions (no target)
├── Later: Gradually tighten

If tROAS is too aggressive:
├── Lower tROAS by 20-30%
├── Or switch to Max Conversion Value
├── Later: Gradually increase

SOLUTION 4: CONSOLIDATE CAMPAIGNS
──────────────────────────────────
Problem: Too many small campaigns
Solution:
├── Combine into fewer campaigns
├── Or use Portfolio bid strategy
├── Share data across campaigns
└── More signals for AI

Example:
├── 5 campaigns x 6 conv/month = Learning limited
├── 1 campaign x 30 conv/month = Better learning
```

### Higher-Funnel Conversion Strategy

```
FUNNEL-UP STRATEGY FOR LEARNING
═════════════════════════════════

WHEN TO USE:
├── <50 primary conversions/month
├── Structurally learning limited
├── Long sales cycle

HOW IT WORKS:
──────────────

STEP 1: Identify Higher-Funnel Action
┌────────────────────────────────────────────────────────────┐
│ Primary Conv    │ Higher-Funnel Option       │ Volume x     │
├─────────────────┼──────────────────────────┼──────────────┤
│ Purchase        │ Add to cart              │ ~5x          │
│ Form submit     │ Form start / Page view   │ ~10x         │
│ Demo request    │ Pricing page view        │ ~20x         │
│ Phone call      │ Click-to-call            │ ~8x          │
│ Signup          │ Email capture            │ ~3x          │
└─────────────────┴──────────────────────────┴──────────────┘

STEP 2: Implement as Secondary Conversion
├── Add conversion tracking
├── First as "secondary" (collect data)
├── Check correlation with primary conversion
└── After 2-4 weeks: switch as primary (for bidding)

STEP 3: Monitor & Adjust
├── Learning should start
├── Check that primary conv doesn't decline
├── Adjust bids for new conversion rate
└── Consider value-based bidding with values

IMPORTANT:
Keep tracking the primary conversion!
Switch back once sufficient volume is reached.
```
